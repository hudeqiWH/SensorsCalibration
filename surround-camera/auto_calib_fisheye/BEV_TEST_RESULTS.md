## 可能的问题和解决方案

### 情况1：交通线完全错位或看不到

**可能原因**：
- BEV投影参数（KG）不适合这些相机
- Tail大小设置不当，裁剪过多
- 外参转换方向错误

**解决方案**：
1. 调整KG参数（src/optimizer.cpp:464-474）：
   ```cpp
   // 尝试不同的地面分辨率
   double ground_resolution = 0.005;  // 0.5 cm/像素（更高分辨率）
   // 或
   double ground_resolution = 0.02;   // 2 cm/像素（更低分辨率）
   ```

2. 调整Tail大小（src/optimizer.cpp:512-518）：
   ```cpp
   // 增大tail以包含更多内容
   sizef = 500;  // 原为340
   sizel = 550;  // 原为390
   sizeb = 530;  // 原为380
   sizer = 550;  // 原为390
   ```

3. 检查外参转换（src/optimizer.cpp:1312-1335）

### 情况2：交通线大致对齐但有明显错位

**可能原因**：
- 初始外参不够精确
- 相机高度提取有误差

**解决方案**：
1. 运行完整标定流程，让算法优化外参
2. 检查相机高度是否正确（当前从外参提取，应该正确）

### 情况3：交通线完美对齐

**下一步**：
1. 运行完整测试：`./test_ocam_debug.sh`
2. 观察Loss值是否下降
