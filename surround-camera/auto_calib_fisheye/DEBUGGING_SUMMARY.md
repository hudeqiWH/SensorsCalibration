## 🔍 可能原因分析

### 原因1: 外参理解错误（高概率）

**问题**: 外参文件格式理解可能有误

**证据**:
- 外参文件中的translation z值都是负的（-0.255, -0.137等）
- 这意味着相机在地面下方，物理上不合理
- 相机应该安装在车辆上方（z应为正值）

**验证方法**:
```bash
# 检查外参文件
 cat /home/nio/deqi/r_calib/data/trans/camera_extrinsics_ikalibr.json5
```

**可能的修正**:
1. translation的符号可能需要反转：t = -translation_vec
2. 或者rotation的方向需要反转
3. 或者外参是body-to-camera而非camera-to-body

### 原因2: 图像旋转方向错误（中概率）

**问题**: Ocam图像旋转方向可能不正确

**当前代码**:
```cpp
rotate(imgf, imgf, ROTATE_180);  // Front
rotate(imgl, imgl, ROTATE_90_COUNTERCLOCKWISE);  // Left
rotate(imgr, imgr, ROTATE_90_CLOCKWISE);  // Right
rotate(imgb, imgb, ROTATE_180);  // Back
```

**验证方法**:
1. 保存旋转后的图像
2. 手动检查交通线方向是否正确
3. 尝试不同的旋转组合

**测试代码**:
```cpp
imwrite("debug_front_rotated.png", imgf);
imwrite("debug_left_rotated.png", imgl);
// ... etc
```

### 原因3: Ocam投影函数数学错误（中概率）

**问题**: `distortPointsOcamFull`中的数学实现可能有误

**可疑代码**:
```cpp
double norm = sqrt(x * x + y * y);  // 只在xy平面归一化
double theta = atan(z_omni / norm);
```

**可能的问题**:
1. 应该使用3D范数：norm = sqrt(x*x + y*y + z*z)
2. 入射角计算方式可能不正确
3. 坐标系转换（标准到Ocam）可能有误

**参考实现**: `ocam_disorder.py:88-119`

### 原因4: 坐标系定义错误（低概率）

**问题**: 相机/地面坐标系定义可能与假设不符

**假设的坐标系**:
- 相机坐标系: z向前，x向右，y向下
- 地面坐标系: z向上，x向前，y向左

**可能的问题**:
1. 相机坐标系z轴可能向上而非向前
2. 地面坐标系可能不同
3. Ocam坐标系定义可能不同
